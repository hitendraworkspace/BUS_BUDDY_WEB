1. PROJECT OVERVIEW
================================================================================

Bus Buddy is a comprehensive school bus tracking and fleet management platform
designed to ensure student safety and streamline transportation operations. The
project consists of:

- Frontend: Next.js 14 web application with TypeScript
- Backend: Supabase (PostgreSQL database with real-time capabilities)
- Python Scripts: Alarm tracking and GPS data processing
- Target Users: Schools, Parents, Drivers, Administrators

Key Features:
- Real-time GPS tracking of school buses
- Live video streaming (MDVR integration)
- Safety alarms and alerts (DMS/ADAS)
- Route management
- Driver management
- Comprehensive reporting
- Multi-platform support (Web, Android, iOS)

7. PYTHON SCRIPTS ANALYSIS
================================================================================

7.1 alarm.py
Purpose: Poll external API for vehicle alarms and upload to Supabase

Key Features:
- Connects to external tracking server (34.56.58.93)
- Authenticates with username/password
- Polls vehicle alarms every 10 seconds
- Groups alarm pairs (e.g., 620+670 = PHONE_USAGE)
- Parses timestamps from alarm descriptions
- Uploads grouped alarms to Supabase

Alarm Types Supported:
- PHONE_USAGE (620, 670)
- CAMERA_BLOCKING (734, 784)
- Driver_Fatigue (618, 668)
- Driver_Distraction (624, 674)
- Smoking (622, 672)
- Harsh_Braking (721, 771)
- Front_Predetection (606, 656)
- Lane_Departure (602, 652)
- Coolide/Collision (600, 650)
- park (247, 297)

Configuration:
- BASE_URL: http://34.56.58.93
- DEVICE_ID: 202600002179
- POLL_INTERVAL: 10 seconds
- PAIRING_WINDOW_SECONDS: 30 seconds

7.2 debug_raw_status.py
Purpose: Debug tool to view raw alarm JSON responses

Features:
- Connects to same API as alarm.py
- Prints full JSON response when alarms exist
- Helps diagnose alarm format issues
- Polls every 1 second for rapid debugging

7.3 guid_tracker.py
Purpose: Track alarm GUIDs to identify new vs existing alarms

Features:
- Maintains set of seen alarm GUIDs
- Marks alarms as NEW or SEEN
- Helps prevent duplicate processing
- Shows alarm metadata (type, speed, parking time)
