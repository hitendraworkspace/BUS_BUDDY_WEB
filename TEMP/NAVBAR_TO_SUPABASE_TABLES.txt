------------------------------------------------------------------------
TABLE 1: bus_routes
------------------------------------------------------------------------
  Columns:
    - id              : number (Primary Key, auto-increment)
    - route_name      : string
    - bus_number      : string
    - start_point     : string
    - end_point       : string
    - device_id       : string (links to GPS tracking device)
    - total_stops     : number
    - estimated_time  : string (e.g., "30 min")
    - status          : string ('active' | 'maintenance' | 'inactive')
    - created_at      : string (timestamp, optional)
    - updated_at      : string (timestamp, optional)

  Used by: Dashboard, Tracking, Routes, Reports, Stream

  Operations:
    - SELECT  → getRoutes() - ordered by route_name
    - INSERT  → upsertRoute() - add new route
    - UPDATE  → upsertRoute() - edit existing route
    - UPDATE  → updateRouteStatus() - change status only
    - DELETE  → deleteRoute() - remove route by id

------------------------------------------------------------------------
TABLE 2: gps_data
------------------------------------------------------------------------
  Columns:
    - id              : number (Primary Key, optional)
    - device_id       : string (links to bus_routes.device_id)
    - latitude        : number
    - longitude       : number
    - speed           : number
    - gps_time        : string (timestamp)
    - created_at      : string (timestamp, optional)

  Used by: Tracking, Routes

  Operations:
    - SELECT        → getGpsHistory() - ordered by gps_time DESC
    - SELECT        → getLatestGps() - latest single record per device
    - REALTIME      → Supabase Realtime subscription on INSERT events
                      (used by Tracking page for live updates)

------------------------------------------------------------------------
TABLE 3: alarms
------------------------------------------------------------------------
  Columns:
    - id              : number (Primary Key)
    - device_id       : string (links to bus_routes.device_id)
    - type            : string (e.g., 'phone', 'fatigue', 'collision',
                        'speeding')
    - description     : string (optional)
    - severity        : string (optional, e.g., 'critical', 'high',
                        'medium', 'low')
    - status          : string (optional, e.g., 'active')
    - arm_date        : string (optional, alarm date)
    - arm_time        : string (optional, alarm time)
    - created_at      : string (timestamp, optional)

  Used by: Dashboard, Alerts, Alarms, Reports

  Operations:
    - SELECT  → getAlarms(limit, deviceId) - ordered by created_at DESC
    - Filterable by device_id (for driver-specific alarms)

------------------------------------------------------------------------
TABLE 4: drivers
